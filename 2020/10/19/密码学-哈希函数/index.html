Basic concepts

Definition

A hash function (with output length ℓ\ellℓ) is a pair of probabilistic polynomial-time algorithms (Gen,H)(Gen,H)(Gen,H) satisfying the following:

  • GenGenGen is a probabilistic algorithm that takes as input a security parameter 1n1^n1n and outputs a key sss. We assume that 1n1^n1n is implicit in sss.
  • HHH takes as input a key sss and a string x∈{0,1}∗x\in \{0,1\}^*x∈{0,1}∗ and outputs a string Hs(x)∈{0,1}ℓ(n)H^{s}(x)\in \{0,1\}^{\ell(n)}Hs(x)∈{0,1}ℓ(n) (nnn is the security parameter implicit in sss).

In general case, the function takes as input strings of arbitrary length. If HsH^sHs is defined only for inputs x∈{0,1}ℓ′(n)x\in\{0,1\}^{\ell '(n)}x∈{0,1}ℓ′(n) and ℓ′(n)>ℓ(n)\ell '(n)>\ell(n)ℓ′(n)>ℓ(n), then we say that (Gen,H)(Gen,H)(Gen,H) is a fixed-length hash function for inputs of length ℓ′\ell 'ℓ′. In this case, HHH is also called a compression function.

First we define an collision-finding experiment Hash−collA,π(n)Hash-coll_{\mathcal{A},\pi}(n)Hash−collA,π​(n) for a hash function Π=(Gen,H)\Pi=(Gen,H)Π=(Gen,H), an adversary A\mathcal{A}A and a security parameter nnn:

  1. A key sss is generated by running Gen(1n)Gen(1^n)Gen(1n).
  2. The adversary A\mathcal{A}A is given sss and outputs x,x′x,x'x,x′.
  3. The output of the experiment is defined to be 1 if and only if x≠x′x\ne x'x​=x′ and Hs(x)=Hs(x′)H^s(x)=H^s(x')Hs(x)=Hs(x′). In such a case we say that A\mathcal{A}A has found a collision.

A hash function Π=(Gen,H)\Pi=(Gen,H)Π=(Gen,H) is collision resistant if for all probabilistic polynomial-time adversaries A\mathcal{A}A there is a negligible function neglneglnegl such that

Unkeyed hash functions

Cryptographic hash functions used in practice generally have a fixed output length and are usually unkeyed, meaning that the hash function is just a fixed function H:{0,1}∗→{0,1}ℓH:\{0,1\}^*\rightarrow \{0,1\}^\ellH:{0,1}∗→{0,1}ℓ. Although this is problematic theoretically since HHH must have a collision pair, the unkeyed cryptographic hash functions used in the real world are collision resistant for all practical purposes. It is due to the fact that colliding pairs are unknown and computationally difficult to find even though they must exist.

In proofs of security for the construction based on collision resistance of a hash function, we just assume that any efficient ways to find a collision in HHH can be used by the adversaries.

Weaker notions of security

In some applications, it suffices to rely on security requirements weaker than collision resistance. These include:

  • Second-preimage or target-collision resistance: if given sss and a uniform xxx, it is infeasible for a PPT adversary to find x′≠xx'\ne xx′​=x such that Hs(x′)=Hs(x)H^s(x')=H^s(x)Hs(x′)=Hs(x). Any hash function that is collision resistant is also second preimage resistant.
  • Preimage resistance: if given sss and a uniform yyy, it is infeasible for a PPT adversary to find a value xxx such that Hs(x)=yH^s(x)=yHs(x)=y (this essentially means that HsH^sHs is one-way).

Domain extension: the Merkle-Damgard transform

Hash functions are often constructed by first designing a collision-resistant compression function handling fixed-length inputs, and then using domain extension to handle arbitrary-length inputs. The Merkle-Damgard transform is a common approach for extending a compression function to a full-fledged hash function, while maintaining the collision-resistance property of the former.

Let (Gen,h)(Gen,h)(Gen,h) be a fixed-length hash function for inputs of length 2n2n2n and with output length nnn. Construct hash function (Gen,H)(Gen,H)(Gen,H) as follows:

  • GenGenGen: a probabilistic algorithm that takes as input a security parameter 1n1^n1n and outputs a key sss
  • HHH: on input a key sss and a string x∈{0,1}∗x\in \{0,1\}^*x∈{0,1}∗ of length L<2nL<2^nL<2n, do the following:
    1. Set B=⌈Ln⌉B=\lceil \frac{L}{n}\rceilB=⌈nL​⌉ (i.e., the number of blocks in xxx). Pad xxx with zeros so its length is a multiple of nnn. Parse the padded result as the sequence of n-bit blocks x1,…,xBx_1,\dots,x_Bx1​,…,xB​. Set xB+1=Lx_{B+1}=LxB+1​=L, where LLL is encoded as an n-bit string.
    2. Set z0=0nz_0=0^nz0​=0n (this is also called the IV).
    3. For i=1,…,B+1i=1,\dots,B+1i=1,…,B+1, compute zi=hs(zi−1∣∣xi)z_i=h^s(z_{i-1}||x_i)zi​=hs(zi−1​∣∣xi​).

Generic attacks on hash functions

Birthday attacks

The collision-finding algorithm we have described is often called a birthday attack.The birthday problem is the following: if qqq people are in a room, what is the probability that two of them have the same birthday? (Assume birthdays are uniformly and independently distributed among the 365 days of a non-leap year.)

For y1,…,yqy_1,\dots,y_qy1​,…,yq​ chosen uniformly in {1,…,N}\{1,\dots,N\}{1,…,N}, the probability of a collision is roughly 1/2 when q=Θ(N1/2)q=\Theta(N^{1/2})q=Θ(N1/2). So, for a hash function to resist collision-finding attacks that run in time TTT, the output length of the hash function needs to be at least 2log⁡T2\log T2logT bits.

Small-space birthday attacks

The small-space birthday attack has similar time complexity and success probability as birthday attacks, but require drastically reduced memory requirements (use constant memory). The attack begins by choosing a random value x0x_0x0​ and then computing xi:=H(xi−1)x_i:=H(x_{i-1})xi​:=H(xi−1​) and x2i:=H(H(x2(i−1)))x_{2i}:=H(H(x_{2(i-1)}))x2i​:=H(H(x2(i−1)​)) for i=1,2,…i=1,2,\dotsi=1,2,…. In each step the values xix_ixi​ and x2ix_{2i}x2i​ are compared; if they are equal then there is a collision somewhere in the sequence x0,x1,…,x2i−1x_0,x_1,\dots,x_{2i-1}x0​,x1​,…,x2i−1​.

Additional application of hash functions

  • Fingerprinting & Deduplication:

    When using a collision-resistant hash function HHH, the hash of a file H(x)H(x)H(x) can be used as a unique identifier of the file xxx. One can check the hashes of two files to determine whether they are equal. This idea is used in many applications like virus fingerprinting, file deduplication, P2P file sharing, etc.

  • Password hashing:

    The user’s passward pwpwpw is stored by a hash of the password H(pw)H(pw)H(pw) instead of a password itself. To prevent exhaustive decryption from an attacker, one way is to use “slow” hash functions (i.e., compute HI(pw)H^{I}(pw)HI(pw) for I>>1I>>1I>>1), and another way is to introduce a salt (i.e., a long random value sss unique to the user)

  • Key derivation:

    A key-derivation function provides a way to obtain a uniformly distributed string from any distribution with high (computational) min-entropy.

  • Commitment scheme:

    A commitment scheme allows one party to “commit” to a message mmm by sending a commitment value comcomcom, with comcomcom reveals nothing about mmm and comcomcom cannot later “open” as two different messages. To commit to a message mmm, the sender chooses uniform r∈{0,1}nr \in \{0, 1\}^nr∈{0,1}n and outputs com:=H(m∣∣r)com := H(m||r)com:=H(m∣∣r).

Practical applications

Hash function from block ciphers

It is possible to build a collision-resistant compression function from a block cipher. One of the most common ways is via the Davies-Meyer construction.

Let FFF be a block cipher with nnn-bit key length and ℓ\ellℓ-bit block length. The compression function h:{0,1}n+ℓ→{0,1}ℓh:\{0,1\}^{n+\ell}\rightarrow \{0,1\}^{\ell}h:{0,1}n+ℓ→{0,1}ℓ is defined as h(k,x)=Fk(x)⊕xh(k,x)=F_{k}(x)\oplus xh(k,x)=Fk​(x)⊕x.

If FFF is modeled as \textit{an ideal cipher}, then the Davies–Meyer construction yields a collision-resistant compression function. However, when instantiating the Davies-Meyer construction with concrete block ciphers, the ciphers should be designed specially for this purpose (block ciphers designed for encryption like DES and AES should not be used).

MD5

MD5 is a hash function with a 128-bit output length. Today, the collisions in MD5 can be found within one minute. So MD5 should not be used anywhere cryptographic security is needed.

SHA family

The Secure Hash Algorithm (SHA) refers to a series of cryptographic hash functions standardized by NIST.

SHA-1 has a 160-bit output length. Theoretical analysis over the past few years indicates that collisions in SHA-1 can be found using significantly fewer than the 2802^{80}280 hash-function evaluations, and it is conjectured that a collision will be found soon. So it is recommended to be replaced by more secure hash algorithms. The details of SHA-1 can refer to: https://blog.csdn.net/cmqwan/article/details/82414954

SHA-2 is a group of hash functions includes SHA-224, SHA-256, SHA-384, SHA-512, etc. SHA-2 is recommended to use in applications. The principle of SHA-256 can refer to: https://blog.csdn.net/jerry81333/article/details/78566418.

SHA-3(Keccak) was chosen as the next-generation replacement of SHA-2. The structure of SHA-3 is very different from SHA-1 and SHA-2. Refer to the following pages for details: https://blog.csdn.net/chengqiuming/article/details/82819769, https://blog.csdn.net/BlackNight168/article/details/82952489
